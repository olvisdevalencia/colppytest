¿Qué solución propondrías a este problema?

Pregunta 2

Tenemos una tabla de 20MM de registros, que crece diariamente. Sobre la misma se realizan además, múltiples consultas para extracción de reportes. En muchos de estos casos, el resultado se demora o bien generan timeout.

¿Que opciones planteas para resolver este problema?

Respuesta 1

  • Revisar los logs al consultar el web service, para detectar posibles problemas.
  • Optimizar la consulta al web service, de manera solo traer los datos necesarios y no todo al mismo tiempo.
  • Limitando la cantidad de peticiones.
  • Verificando que se halla cerrado la peticion una vez realiazada la misma.
  • Una vez hecho esto y detectado el problema se puede reaccionar a un posible bug fix del mismo.

Respuesta 2

  • Revisar los metodos de la clase generadora de la consulta.
  • Optimizar la consulta a la base de datos, muchas veces se traen datos o campos que no son utilizados.
  • Esto tambien tiene mucho que ver con la memoria del servidor, performance del query.
  • Revisar que este bien diseñada y normalizada la base de datos es uno de los factores claves,ya que dan mas que todo este tipo de timeout sino estan bien optimizadas.
  • Una de las cosas que me ha resultado util es separar la logica el front y el backend.
  • De tal manera que el front solo quede realizando las peticiones y del lado del backend lo mismo.
  • Es decir balanceando la carga entre ambas, solo se usara el front para presentacion y peticion de los datos.
  • Dejando la carga de la base de datos aparte, manejando esta informacion con alguna utilizando JSON, compresión, y una API RESTFul.
